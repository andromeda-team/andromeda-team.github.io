Raw JSON Text Format

Is a versatile and powerful text format used in minecraft commands and files. Just name a few:

  • /tellraw command;
  • /say command (op only);
  • /title command;
  • *.mcmeta files;
  • entity/item names;
  • item description;

In case of commands, it's a one-line JSON interpretation, so for better understanding it is recommended to understand basics of how JSON Format works.

JSON Format basics

JSON is based on a key-value system, which is simply key:value - For every key (a string) there's a value, which can be an Object, an Array, an Integer, a String and so on.

As an example, if you want to give our text a color we could do "color":"green".

In case of arrays/objects we could do: "my_pets":["Jean", "Paul"] and so forth.

In minecraft there are different keys mojang gave us access to, as well as coresponding list of valid values that we can use.

The most important part, in order to use a string you must wrap it into double quotes "", but for JSON to work you must wrap everything in curly braces {}.

Plain text

If we want to say Hello World! in chat, what we could do is use /tellraw @a "Hello World!", by simply putting the text into double quotes.

If we want to send the same message but using JSON, we should use /tellraw @a {"text":"Hello World!"}. Here we used the text key, which defines the plain text element. (Note: These two commands will output the same result in chat)

There are some versatile keys, that we can use to beautify our text:

  • "color":"VALUE" changes color of the text. You can replace VALUE with:
    • dark_red, red, gold, yellow, dark_green, green, aqua, dark_aqua, dark_blue, blue, light_purple, dark_purple, white, gray, dark_gray, black
    • reset Resets the color to default one
    • #XXXXXX (HEX Color Format) In versions 1.16+ there's a possibility to use HEX Color Format for any custom color. (example: #454545)
  • "bold":VALUE sets whether the text is bold. You can replace VALUE with:
    • true
    • false
  • "italic":VALUE sets whether the text is italic. You can replace VALUE with:
    • true
    • false
  • "underlined":VALUE sets whether the text is underlined. You can replace VALUE with:
    • true
    • false
  • "strikethrough":VALUE sets whether the text is strikethrough. You can replace VALUE with:
    • true
    • false
  • "obfuscated":VALUE sets whether the text is obfuscated. This means that all symbols will replace with a random character making text completely unreadable. You can replace VALUE with:
    • true
    • false

As well as behavior controlling keys:

  • "clickEvent":{"action":"ACTION", "value":"VALUE"} sets the event to occur when text is clicked by player (only works in chat). ACTION can be set to:
    • run_command Despite the name of this value it sends a message as player who clicked the text. In order to run a command just use / in the very beginning.
    • suggest_command Replaces player's chat input field with a specified string. Extremely useful for user input suggestions.
    • open_url Opens specified url in player's browser (displays a confirm window first)
    • copy_to_clipboard [1.15+] Copies specified string into player's system clipboard.
    • change_page Works only in books, changes the page.
  • "hoverEvent":{"action":"ACTION", "contents":CONTENTS} sets the event to occur when text is hovered by player (only works in chat). ACTION can be set to:
    • show_text Displays text in JSON/String format
    • show_item Displays an item tooltip, as if it was hovered in inventory. Accepts the following parameters:
      • id the id of an item. Example: minecraft:diamond
      • count
      • tag (optional) the tag of an item. Example: {display:{Name:'"MY DIAMOND!"'}}
    • show_entity Displays an entity's name, type, and UUID. Accepts the following parameters:
      • id the UUID of an entity.
      • type the type of an entity. Example: minecraft:creeper
      • name (optional)
