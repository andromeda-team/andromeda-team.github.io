Description

Storages are extremely powerful, though they create the same latencies as any other /data commands.

Since storages can store any data in any paths and names you wish, they can be used to create a "database" for players/other.

The reason you would prefer a "database" over using scoreboards for players is because you can't query score of an offline player, this isn't true for our databases!

Entity database

In this section we will cover creating and managing entity database, we will use players, though it will work for any entities!

Setting properties (optional) You can set variables to your personal ones, to get the best experience out of explanation. You will also be able to simply copy & paste the code into the corresponding functions!

First, we need to register player into the database. To do so we can create a function:

  • #! Function :player/register

  • tag @s add -.in_database

  • # UUID is used to link this entry to an entity. You can add any other properties you wish. I will simply add a Score
  • data modify storage minecraft: Database append value {UUID:[I; 0, 0, 0, 0], Score:0}
  • data modify storage minecraft: Database[-1].UUID set from entity @s UUID

This will create a blank entry, that has a score.

API

Since we are working with a custom data pack, we will need to create an "API". Interface, that we will use to access it, since storages aren't directly connected to an entity.

I recommend create a separate directory for this. For instance I will use /data/namespace/functions/api/database/.

Inside we will have /read/ and /write/ directories, to make calls more readable.

Read API

The simplest one will be reading from the database
All we need to do is parse through the database and return the one that has the same UUID as the player

Setting properties (optional) You can set variables to your personal ones, to get the best experience out of explanation. You will also be able to simply copy & paste the code into the corresponding functions!

Create a function to prepare database for reading:

  • #! Function :api/database/open

  • # Create a copy of the database. We will use the parsing algorithm
  • data modify storage minecraft: Thread set from storage minecraft: Database
  • data modify storage minecraft: Instance set value {}

Let's create a function to get entry by uuid: We can then run, for example: data get storage minecraft: Instance.Score. This will be the score of the player with this UUID

  • #! Function :api/database/read/get_entry_by_uuid

  • function :api/database/open
  • execute if data storage minecraft: Thread[] run function :api/database/read/get_entry_by_uuid/step
  • execute unless data storage minecraft: Instance.UUID run ... (failed)
  • execute if data storage minecraft: Instance.UUID run ... (succeed)

Now the step:

  • #! Function :api/database/read/get_entry_by_uuid/step

  • # We compare UUIDs and if they match we return the result, if they don't we run the function again for the next element
  • execute store success score #Succeed run data modify storage minecraft: Thread[-1].UUID set from entity @s UUID
  • execute if score #Succeed matches 0 run function :api/database/read/get_entry_by_uuid/return
  • data remove storage minecraft: Thread[-1]
  • execute if score #Succeed matches 1 if data storage minecraft: Thread[] run function :api/database/read/get_entry_by_uuid/step

Return function if "succeed":

  • #! Function :api/database/read/get_entry_by_uuid/return

  • data modify storage minecraft: Instance set from storage minecraft: Thread[-1]
  • data remove storage minecraft: Thread

Write API

Writing is more complicated, because even though we can remove/insert certain elements from/to an array, we can only do so for static values like "4", when we want it to work for elements that have the same ID. Thus, we are going to use parsing

Setting properties (optional) You can set variables to your personal ones, to get the best experience out of explanation. You will also be able to simply copy & paste the code into the corresponding functions!

As an example, we want to set score to a player using UUID. We will store our score inside #Setter

  • #! Function :api/database/write/set_score_by_uuid

  • # We prepare the database for parsing and then erasing the current database
  • scoreboard players set #Setter 1337
  • function :api/database/open
  • execute store result score #Counter run data get storage minecraft: Thread
  • data remove storage minecraft: Database
  • execute if score #Counter matches 1.. run function :api/database/write/set_score_by_uuid/step

The step:

  • #! Function :api/database/write/set_score_by_uuid/step

  • scoreboard players remove #Counter 1

  • # We must copy it each time, because in case with writing, we do not want to modify storage that doesn't have the same ID as we're searching for!
  • data modify storage minecraft: Instance set from storage minecraft: Thread[-1].UUID

  • execute store success score #Succeed run data modify storage minecraft: Instance set from entity @s UUID
  • execute if score #Succeed matches 0 run function :api/database/write/set_score_by_uuid/set

  • # After (not) modifying we just append the value back to the database and repeat the cycle until we're done
  • data modify storage minecraft: Database append from storage minecraft: Thread[-1]
  • data remove storage minecraft: Thread[-1]

  • execute if score #Counter matches 1.. run function :api/database/write/set_score_by_uuid/step

We set the Score:

  • #! Function :api/database/write/set_score_by_uuid/set

  • # In our case we could just put the following line without creating a function, but in practice, when you want to modify more than one value having a function is better.
  • execute store result storage minecraft: Thread[-1].Score int 1 run scoreboard players get #Setter

This will set the score of a player who executes the main function to 1337. Cool, isn't it?
